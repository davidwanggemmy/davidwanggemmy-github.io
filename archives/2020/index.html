初识 git

git 的版本控制

  • git 并不是基于差异的版本控制，相反，每当你提交更新或保存项目状态的时候，git基本上就会对当时全部的文件创建一个快照并保存这个快照的索引，如果文件没有修改,git 不再重新存储该文件，而是只保留一个链接指向之前存储的文件

git 的三种状态

  • 已修改：表示修改了文件，但还没保存到数据库中
  • 已暂存：表示对一个已修改文件的当前版本做了标记，使之包含在下次提交的快照中．
  • 已提交：表示数据已经安全地保存在本地数据库中
  • 对应git项目拥有三个阶段：工作区 , 暂存区　, git 目录

安装 git

  • sudo apt install git-all

  • 用户信息

    • 安装完 Git 之后，要做的第一件事就是设置你的用户名和邮件地址。 这一点很重要，因为每一个 Git 提交都会使用这些信息，它们会写入到你的每一次提交中，不可更改：

    • $ git config --global user.name "lq-debuger"
$ git config --global user.email lqe@example.com
